实践经验
======
**Crowd Tracking: 视频人物追踪与轨迹可视化**
* 重建了一个计算机视觉演示项目，支持常见视频格式上传，使用 YOLO 检测人物，并通过 DeepSORT 在每一帧维护人物身份。
* 实现了 **FastAPI** 后端，可生成标注后的视频、轨迹小图、JSON/CSV 追踪结果和任务摘要，方便本地复盘。
* 准备了 Docker、Cloud Build、Postman 和部署配置，用于低成本 **Google Cloud Run** 作品集服务。

**Twitch+: 个性化的 Twitch 资源推荐引擎**
* 设计并构建了一个部署到 **AWS App Runner** 的全栈 **Spring Boot** 应用程序，供用户搜索 Twitch 资源并获得推荐，以及一个使用 **React** 和 **Ant Design** 的用户友好的前端。
* 实施了 **RESTful API** 以通过 **Twitch API** 与 **OpenFeign** HTTP 客户端检索真实的 Twitch 资源，并使用 **Spring Security** 管理用户身份验证（注册/登录/注销）。
* 从 Twitch 资源中提取游戏信息以实现基于收藏夹的基于内容的推荐解决方案，利用 **AWS RDS** 上的 **MySQL**，**Spring Data JDBC**，并通过 **Caffeine** 缓存优化性能。

**Staybooking: 在线住宿租赁应用程序**
* 使用 **React** 和 **AntD** 作为前端架构单页 Web 应用程序，由基于 **Spring Boot** 的后端服务支持，以管理住宿上传、删除、搜索和预订。
* 利用 **PostgreSQL** 存储用户生成的数据和预订历史记录，集成 **Google Cloud Storage** 以存储媒体文件，并将高度可扩展的后端服务部署到 **Google Cloud Run**。
* 利用 **Spring Security** 框架实施了基于令牌的服务器端安全用户身份验证。

**OnlineOrder: 基于 Spring Boot 的在线点餐系统**
* 使用 **Spring Boot** 架构了在线点餐系统，具有清晰的控制器、服务、存储库分离结构，并利用依赖注入来增强可维护性和可扩展性。
* 使用 **React** 和 CRUD **REST API** 控制器构建前端，包含注册、菜单搜索、订购和结账等功能。部署在 **AWS** 上以实现高效部署。
* 开发了带有 **PostgreSQL** 数据库的 **Spring Data JDBC** 并托管在 **AWS RDS** 上，处理与菜单、购物车和餐厅运营相关的数据。

**SocialAI: 基于 AI 的社交网络**
* 实施了一个使用 **React** 的社交网络平台，专注于用户友好的设计和无缝交互。利用 **OpenAI 的 Dall-E 3** 协助用户创建和更新帖子。
* 使用 **Go** 中可扩展的 Web 服务来处理帖子，并使用 **ElasticSearch** (部署到 **Google App Engine**) 提供搜索功能，供用户搜索最近的帖子。
* 使用带有 **React Router v4** 的基于令牌的注册/登录/注销流程和带有 **JWT** 的服务器端用户身份验证改进了身份验证。

**Next AI: 基于 Web 的全栈交互式 AI 代理，用于 PDF 文档查询**
* 利用 **React** 和 **Ant Design** 架构交互式对话 UI，使用户能够轻松地实时上传 PDF 文档并与之交互。
* 通过 **Express** 和 **Node.js** 架构和实现了 **RESTful API**，针对高性能请求处理进行了优化。
* 集成 **OpenAI 的 GPT-3.5 Turbo API** 和 **LangChain** 技术，实现了一个能够进行文档上传、拆分、存储、检索和输出功能的复杂 AI 代理。

**Mini Spotify: 移动音乐应用程序**
* 充分利用 **Android Jetpack Library** 和 **Hilt Dependency Injection** 设计了一个 Kotlin Spotify 应用程序。
* 使用 **Jetpack Navigation** 组件实现了 BottomBar 和应用程序导航，使用 json-server 创建了 **RESTful API**，并使用 **Retrofit** 处理请求。
* 使用 **Jetpack Compose** 遵循 MVVM 架构构建了 feed/album/favorite UI，并通过使用 **Room Database** 为最喜欢的音乐功能启用了本地缓存功能。
